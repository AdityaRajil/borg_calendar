Remote Server

 

The Remote Server functionality allows BORG to access its database through a connection to a servlet running in a servlet container. It has been tested with TOMCAT.

 

The servlet provides an XML over HTTP interface to the BORG client program. It does not provide browser based web access.

 

The main features provided by the remote server are:

  • Remote access
  • Secure access over SSL, suitable for access over the internet
  • Multi-User authentication
  • Multi-User access to a single database with separate calendars for each user.

 

Note: For a single user who runs BORG on their local machine, the remote server configuration does not provide any advantages over the simple file-based database or a normal MySQL configuration. The remote server is primarily for users wanting the features listed above.

Setup

 

Some knowledge of Tomcat is assumed.

 

  1. Install Tomcat (http://tomcat.apache.org/) on the machine that will be the data server. The database will also reside on the same machine. A MySQL database is strongly recommended for use with the remote servlet, although the servlet is able to use the default file-based database.
  2. If you are using MySQL, set up your database. Instructions can be found elsewhere.
  3. Install BORG 1.5.1 beta or later – and make sure to install the optional remote server package.
  4. Start Tomcat and deploy the web application. The web application can be found in the folder where BORG was installed. It is called borg_servlet.war. One easy way to deploy is to copy borg_servlet.war to TOMCAT_HOME/webapps (where TOMCAT_HOME is the top folder of the tomcat install). When deployed, a borg_servlet folder will be created in the TOMCAT_HOME/webapps folder.
  5. Shutdown Tomcat and edit the file: TOMCAT_HOME/webapps/borg_servlet/WEB-INF/web.xml. In this file, change the param value of the dburl parameter to be either a URL for your  MySQL database or a folder name to hold the plain db files. (MySQL is recommended). Here is an example web.xml file:

 

<?xml version="1.0" encoding="ISO-8859-1"?>

 

<!DOCTYPE web-app

    PUBLIC "-//Sun Microsystems, Inc.//DTD Web Application 2.3//EN"

    "http://java.sun.com/dtd/web-app_2_3.dtd">

 

<web-app>

    <!-- General description of your web application -->

    <display-name>BORG Calendar</display-name>

    <description>BORG Calendar Backend</description>

 

    <servlet>

      <servlet-name>controller</servlet-name>

      <description>BORG Calendar Controller Servlet</description>

      <servlet-class>net.sf.borg.servlet.BORGServlet</servlet-class>

       <init-param>

         <param-name>dburl</param-name>

         <param-value>jdbc:mysql://localhost/borg?user=borg&amp;password=borg&amp;autoReconnect=true</param-value>

       </init-param>

    </servlet>

   

    <servlet-mapping>

      <servlet-name>controller</servlet-name>

      <url-pattern>/app</url-pattern>

    </servlet-mapping>

 

    <security-constraint>

       <web-resource-collection>

          <web-resource-name>BORG Calendar Backend</web-resource-name>

          <url-pattern>/app</url-pattern>

       </web-resource-collection>

       <auth-constraint>

          <role-name>borg_user</role-name>

       </auth-constraint>

    </security-constraint>

   

    <login-config>

       <auth-method>BASIC</auth-method>

       <realm-name>BORG Calendar</realm-name>

    </login-config>

 

</web-app>

 

The highlighted text shows a url that connects to a database on the local machine with user=borg and password=borg (you should use your own values here). Notice that since the URL is embedded in XML, the ampersands in the URL need to be escaped as “&amp;” instead of just a single “&” character. The MySQL user is NOT the BORG end-user, it is the MySQL user id that has access to your BORG database. This URL is used internally by the server to connect to the database. It is not the URL that a user would use to connect to the remote server.

 

  1. Add your borg users (the end-users)  to the tomcat users file – TOMCAT_HOME/conf/tomcat-users.xml. Each borg user should have a role of “borg_user”. Here is an example :

  <role rolename="borg_user" description="borg_user"/>

  <user username="mbb" password="hoohah" fullName="Mike" roles="borg_user"/>

  <user username="ckb" password="doodle" fullName="Cathy" roles="borg_user"/>

  1. Start up Tomcat
  2. Start BORG and under the database tab of the options screen, select the Remote Server radio button. Then enter the following url: http://yourtomcathost:8080/borg_servlet/app. Replace “yourtomcathost” with the machine that you are running tomcat on. If you are using a port other than 8080 for tomcat, change the port in the URL. This is the URL that any user on any machine would use to connect to the remote server.
  3. BORG will prompt for your user name and password each time you start BORG. This data is not kept under the existing db options – there is a new prompt window where you enter this. Enter a user name from the tomcat users file. For example, I would connect to my db using user=mbb password=hoohah.  Multiple users can now connect from any machine with their own usernames and passwords to access their own data. You can run multiple BORGs on the same machine as different users to access different calendars.

 

Security Tips

  • Your tomcat files, especially the usernames and passwords should not be accessible to your users.
  • Your MySQL database user and password should not be known to your users – otherwise, they can connect using any MySQL client and access any user’s data.
  • If you are really running this over the internet, make sure you secure your MySQL database (the MySQL manual covers this). This includes things like not using the root login for borg, securing the root login with a password, only granting the borg db user access to the borg database tables and not access to other dbs. (And as I said above, pick something other than “borg” as your db username and password.)
  • If you are running over an insecure network – such as the internet, you should  use SSL (i.e. https://yourtomcathost:8443/borg_servlet/app) to get a secure connection from the machine running BORG to the machine running the remote server. NOTE: SSL access does not work in the 1.5.1beta release from 9/26 if you roll your own certificate when setting up Tomcat, but this should be fixed shortly.
