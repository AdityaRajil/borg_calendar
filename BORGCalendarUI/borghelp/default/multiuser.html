Multi User Support refers to the support for having multiple users connect to a single database.

 

There are two supported options for multi-user support:

 

  • Remote Server

 

  • MySQL Shared DB

 

Remote Server

 

The remote server  is a separate application that runs inside a Servlet container (i.e. Tomcat). With the remote server, there is a single database running on the server machine (MySQL recommended). Each user has a login and password. Each user runs the same BORG program that they do for other DB types, but they are prompted for a login/password in order to connect to the remote server. Users’ data is all kept in the same database and in the same database tables, but each user only sees their own data.

Since all of the data is in the same tables, this allows for features where users can share data. As of BORG 1.6, the only such feature is shared calendars, where calendars can be marked as shared. A shared calendar can be seen by all users as read-only. This would allow for people to have their own private calendars, plus any number of shared calendars. As of 1.6, a user would have to login as the owner of the shared calendar to make changes to it.

 

Summary:

  • Remote server supports separate data per user in a single remote database.
  • Each user has a login/password.
  • Shared calendars are supported (read-only to all users, writable for the shared calendar owner).

 

 

There is a separate help topic to cover how to set up the remote server. Please refer to that for more information.

 

MySQL Shared DB

 

Multiple users can connect to a single MySQL DB. Any number of independent database logins can be set up in a single MySQL installation.

·        If users connect to different MySQL logins, their BORG data will be completely independent.

·        If users connect to the same MySQL login, they will share a single set of data. In this case, BORG supports a periodic sync with the database so that if one user updates data, the changes will be periodically loaded to the GUIs of all running users. This is the simplest way to have a single shared set of data.

 

Shared-File Limited Multi-User Support for a single database (deprecated)

 

This feature is deprecated since it is based on the Local File database type. It may be de-supported in a future release.

 

A form of limited  multi-user support is in releases 1.3.2 and later to allow users to share a single calendar by simply opening the same database files at the same time. You have to turn it on from the options menu and restart (if you want to use it).

This feature is not “full-blown” multi-user support. This feature simply supports multi-user concurrent access to a single file-based database using files on a shared drive.

In general, the way it works is that all BORG clients sharing the same local DB files will use locking to prevent concurrent access. There will be a counter in the database that each client will use to determine if another client has updated the database since the last time it checked the DB. If a client determines that the DB has been updated by another client, it will flush all cached data and rescan the database to sync up.

A menu option has been added to force BORG to sync with the database. This is useful when you want to pull in other users' changes but have no desire to update the DB yourself. BORG will always sync the DB before it tries to update it, but if you are just browsing data, BORG will run from its cached data and not force a sync - hence the sync menu option in case you want to pick up someone else’s changes.
