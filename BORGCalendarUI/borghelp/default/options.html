Options

Options can be changed from the Options menu. 

 

FYI – On some systems, the options screen does not appear instantly (like other screens). A delay is normal and is caused by the lookup (by the Java runtime) of the list of  “Look and Feels” installed on the system.


Most of the options from the options screens are self-explanatory - except:

 

Appearance

 

  • Look And Feel - the combo box will be populated only with the installed look and feels in your JRE and also the Jgoodies look and feels from jgoodies.com. Some of the L&Fs from Java are not so appealing with BORG. The PlasticXP look and feel is distributed with BORG to make sure you have at least one good looking one. If you change the look and feel, restart BORG to have it take effect.

  • Sort Appointments by Color – normally appointments on the same day are sorted by time. If this option is selected, the appointments are sorted first by color – red, blue, green, black, white. This allows high-priority appointments (i.e. red) to move to the top of the visible list for a day.

  • Week View Start and End Hour – the range of times shown on both the Week and Day View grids.

 

  • Holidays - Most US and Canadian holidays can be shown (optionally). Except for Christmas, religious/ethnic holidays are not yet included - mainly because they are usually lunar based and much harder to calculate - and because they aren't usually a day off from work. These can be added manually through the appointment editor.

 

  • Wrap Appointment Text - this will cause text to wrap in the month view and print. To take effect in the month view, the window must be destroyed and recreated either be restarting BORG.

 

  • Truncate Appointments in Month View – this is set by default and causes only the first line of appointment text to show in the month view. If you want all lines of multi-line appointments to show in the month view, shut this feature off.

 

Database Information

 

  • Apply DB Change will restart BORG with the new Database. No data is copied and the original database is not removed. If you just want to move or copy db files to a new location – just copy them and change to the new location from the options screen. For more complicated DB changes – like changing from local files to MySQL, just use import/export to XML to move the data into the new database.

Email Parameters

An email message is sent once a day only, and only for the next day's appointments. Provide the name of an SMTP server and email address to send to.

Emails can also be sent per appointment at a set time before the appointment

 

Printing

 

  • Print Logo - A user defined print logo can be included on the month print. The logo space will be the last two empty boxes on the bottom right. This will have a size of 184x72 pixels. Any logo smaller than this will just be centered in this space. Any logo larger will be scaled to fit - retaining its aspect ratio. The logo can be GIF, JPG, or PNG. A color logo will print in color, even if the color-print option in BORG is turned off.

 

Miscellaneous

 

  • Auto Update Check - this only causes BORG to compare the latest release number from the BORG sourceforge web site to the release that you are running. If the releases are different, a popup will inform you. Nothing else is done. The feature is off by default. The check is done once per day. Only stable release numbers are kept on the web. New Beta releases will not be detected.

 

  • Start in Background - This will cause BORG to start only as a systray icon. No other windows will be opened - except for reminder popups and the auto version check if applicable. BORG will still popup reminders and send email while running in the background. The main month view can be started by double clicking the systray icon or from the systray icon menu.

 

  • Show Stack Trace Option – if an error occurs and this option is set, a button will appear with the error screen to generate a stack trace. This can be used for debugging the error and can be sent to the developers. This need not be turned on unless debugging is being done.

  • Export Todos as Events (iCal) – when exporting to iCal format, todos are exported as VToDo’s. Some calendar programs only display VtoDo’s on a checklist and not in their calendar. Check this option to force to do’s to be exported as regular appointments (VEvents).

  • Reverse the Date Button/Day Text mouse actions – In release 1.4, the way to get the appointment editor was changed from clicking to the date button to clicking the day text area. The date button was then changed to pop up the new Day View. This change was asked for by a few people. If you prefer to click the date button to get the editor, select this option to restore the old behavior.

  • Socket Port – setting a value here, (2929 is the default), will allow the program to detect when another copy is running. If you start a second copy of BORG when one is already running, and the socket port is set, the second copy will just open the running copy and exit. In addition, setting the socket port also allows the Palm Conduits to sync to an HSQLDB database.

 

Popup Reminders – see the separate help page for popups.
